# DataOfis Partners --- Hero Section Specification

## PURPOSE

Transform the hero section from a static centered block into a **layered
editorial entry point** with three functions:

1.  Positioning (headline + framing)
2.  Active thinking (scrolling strip)
3.  System entry (topic selector)

------------------------------------------------------------------------

## STRUCTURE

### 1. Navigation Trigger

-   Small compact block inside hero (top-left)
-   Contains:
    -   hamburger icon
    -   "DataOfis Partners"
-   Not full-width
-   Floating within hero grid

------------------------------------------------------------------------

### 2. Optional Label

-   Text: "DataOfis Partners" or remove
-   Font: 12--14px
-   Uppercase
-   Low contrast

------------------------------------------------------------------------

### 3. Headline

-   Keep existing headline
-   Left-aligned
-   Max width: 640--720px
-   Font size: 48--64px
-   Weight: 600--700
-   Line height: 1.1--1.2

------------------------------------------------------------------------

### 4. Supporting Text

-   Max width: 520--600px
-   Font size: 16--18px
-   Line height: 26--28px

------------------------------------------------------------------------

### 5. Assertion Lines (Replace bullets)

Examples: - End-to-end direction. Not fragmented initiatives. -
Senior-level ownership. Not delegated execution. - Accountability for
outcomes. Not activity.

Spacing: - 12--16px between lines - Font size: 14--16px

------------------------------------------------------------------------

### 6. CTA Button

-   "Schedule Executive Data Review"
-   Positioned below assertions
-   Medium size button

------------------------------------------------------------------------

## DIVIDER

Add vertical spacing (48--64px)

------------------------------------------------------------------------

## 7. FROM OUR WORK (Scrolling Strip)

### Purpose

Signal active thinking and real-world patterns.

### Layout

-   Horizontal scroll container
-   4--6 cards visible (desktop)
-   Scrollable on mobile

### Card Structure

-   Optional label (small)
-   Bold statement (1--2 lines)
-   Supporting line (1 line)

### Example Content

-   Reports do not create value --- decisions do\
    Why organizations stop at insight

-   Data exists. Decisions don't change\
    Systems fail without structure

-   AI initiatives fail before they start\
    Missing prerequisites block execution

### Typography

-   Title: 18--20px, weight 600
-   Body: 14--16px

------------------------------------------------------------------------

## 8. TOPIC SELECTOR

### Purpose

Entry into system structure

### Items

-   Direction
-   Architecture & Design
-   Organization & Control
-   Execution & Adoption

### Layout

-   Horizontal row
-   Centered or grid-aligned
-   Minimal styling (underline or pill)

### Behavior

-   Click → navigate or preview
-   Active state clearly visible

------------------------------------------------------------------------

## LAYOUT SYSTEM

-   Use full-width grid (12 columns)
-   Avoid centered single-column layout
-   Hero text aligned to left grid columns
-   Navigation trigger aligned independently
-   Maintain strong whitespace

------------------------------------------------------------------------

## DESIGN RULES

-   No images in hero
-   No decorative UI
-   Typography-driven
-   Clean spacing
-   Calm and controlled

------------------------------------------------------------------------

## RESPONSIVE

-   Stack elements vertically on mobile
-   Scroll strip becomes swipeable
-   Topic selector becomes scrollable row

------------------------------------------------------------------------

## OUTPUT EXPECTATION

-   Component-based structure
-   Clean separation of:
    -   hero text
    -   strip
    -   selector
-   Grid-based alignment
