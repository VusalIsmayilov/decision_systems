# Design-system language
---

## 1. Core Principle — Structured Intelligence

* Design expresses **logic, not decoration**
* Every element must have a **functional role in comprehension**
* Visual hierarchy reflects **decision hierarchy**
* Content is **deconstructed into units**, not presented as blocks
* Emphasis is driven by **importance, not aesthetics**
* No visual element exists without **semantic justification**
* Clarity is achieved through **reduction and ordering**
* The interface prioritizes **interpretability over engagement**
* Design communicates **how to think, not what to feel**

---

## 2. Layout System — Not Grid, but Logic

* Layout is derived from **information structure**, not symmetry
* Grid is used as a **constraint tool**, not a visual goal
* Each section has a **dominant spatial logic**
* Layout defines **reading sequence explicitly**
* Structural zones are **functionally differentiated**
* Alignment is used to **group meaning**, not decorate
* Composition prioritizes **flow over balance**
* Sections are treated as **independent logical systems**
* No default or reusable “template” layout across sections

---

### A. Section Composition Patterns

* Each section maps to **one primary cognitive model**
* Spatial structure must **visually encode that model**
* Content is **forced into structure**, not adapted loosely
* Patterns are **finite, predefined, and consistently applied**
* Internal hierarchy must be **visually explicit**
* Transitions between sections must **change composition logic**
* Repetition of patterns is **controlled and non-consecutive**
* Structural clarity overrides visual variation

---

### B. Asymmetry Over Balance

* Visual weight is **intentionally uneven**
* Layout avoids **centered or equal distribution by default**
* One element or zone must **dominate attention**
* Secondary elements provide **context, not competition**
* Negative space is used to **create directional tension**
* Imbalance is used to **guide reading flow**
* Symmetry is only used when **conceptually required**

---

## 3. Typography — The Real Design Engine

* Typography defines **structure, hierarchy, and rhythm**
* Font selection is **neutral, precise, and highly legible**
* Hierarchy is enforced through **size, weight, and spacing—not styling**
* Line length is **strictly controlled for readability**
* Text is broken into **atomic units, not continuous blocks**
* Emphasis is achieved through **position and scale, not decoration**
* Vertical rhythm is **consistent and system-driven**
* No expressive or stylistic typography elements
* Typography carries **the majority of visual differentiation**

---

## 4. Color — Minimal, Functional, Intentional

* Color palette is **restricted and controlled**
* Base colors define **structure, not mood**
* Accent color is used **sparingly and consistently**
* Color indicates **function, hierarchy, or state**
* No gradients, decorative effects, or unnecessary variation
* Contrast is optimized for **clarity and accessibility**
* Color usage is **systematic, not contextual or expressive**
* Background and foreground relationships are **stable and predictable**

---

## 6. Spacing — Invisible but Critical

* Spacing defines **relationships between elements**
* Vertical spacing encodes **hierarchy and grouping**
* Larger gaps indicate **conceptual separation**
* Tight spacing indicates **logical cohesion**
* Margins create **focus and containment**
* Spacing is consistent across **similar structural roles**
* No arbitrary spacing—every value is **system-defined**
* Whitespace is used as an **active structural element**

---

## 7. Interaction — Minimal but Intentional

* Interaction supports **navigation and comprehension only**
* Motion is **subtle, controlled, and purposeful**
* No interaction that competes with **content understanding**
* Hover and transition states provide **clarity, not decoration**
* Animation duration and easing are **consistent system-wide**
* No reliance on interaction for **critical information delivery**
* Interactions reinforce **structure and hierarchy**
* Avoid immersive or attention-heavy behaviors

---

## 8. Components That Define the Style

* Components are **functional primitives, not decorative elements**
* Each component has a **clear structural role**
* Components are designed for **reusability across contexts**
* Variations are controlled through **system rules, not ad hoc changes**
* Components support **content decomposition and hierarchy**
* No component exists purely for **visual enhancement**
* Component behavior is **predictable and consistent**
* Composition of components reflects **logical relationships**

---

## 10. Common Mistakes to Avoid

* Treating layout as **visual arrangement instead of logical structure**
* Repeating identical section compositions
* Defaulting to **symmetrical or centered layouts**
* Allowing content to form **undifferentiated text blocks**
* Mixing multiple thinking models within one section
* Overusing color or introducing **non-functional styling**
* Using typography decoratively instead of structurally
* Ignoring spacing as a **hierarchy tool**
* Introducing unnecessary interaction or motion
* Designing components without **clear semantic purpose**
* Letting visual consistency override **structural clarity**
* Optimizing for aesthetics over **cognitive clarity**






# The complete thinking model systems 

---

## 1. Sequence (Order-Dependent Logic)

**What it expresses:**
Things must happen in a specific order.

**Use when:**

* steps
* phases
* system flow
* dependency

**Signal language:**

* first / then / within / after
* operates within
* depends on

**Your example:**

> Direction → Structure → Execution

---

## 2. Causality (Why Something Works or Fails)

**What it expresses:**
Cause → mechanism → outcome

**Use when:**

* explaining necessity
* proving logic
* showing consequences

**Signal language:**

* without X → Y fails
* leads to
* results in
* creates / prevents

**Your earlier section:**

> Execution without direction disperses effort

---

## 3. Constraint (Boundary of Validity)

**What it expresses:**
This only works under certain conditions.

**Use when:**

* defining limits
* removing ambiguity
* enforcing discipline

**Signal language:**

* only when
* cannot exist without
* requires
* condition for

**Example:**

> Control exists only when this order is maintained

---

## 4. Contrast (False vs Correct)

**What it expresses:**
Wrong approach vs correct approach

**Use when:**

* reframing thinking
* breaking assumptions
* positioning your method

**Signal language:**

* not X — but Y
* most organizations… / we…
* instead

---

## 5. Principle (Foundational Truths)

**What it expresses:**
Non-negotiable rules

**Use when:**

* defining your philosophy
* setting system laws

**Signal language:**

* this is not a preference
* always
* never
* must

---

## 6. System (Interdependent Structure)

**What it expresses:**
Multiple elements working together

**Use when:**

* describing architecture
* showing relationships
* explaining how parts connect

**Signal language:**

* operates within
* supports
* connects to
* integrated into

---

## 7. Evaluation (Judgment Framework)

**What it expresses:**
How something should be assessed

**Use when:**

* diagnosing problems
* assessing maturity
* defining quality

**Signal language:**

* measured by
* defined as
* evaluated through

---

## 8. Transformation (State Change)

**What it expresses:**
From → To shift

**Use when:**

* before/after
* capability building
* change narrative

**Signal language:**

* from X to Y
* shift
* evolve
* move from

---






# The complete BCG-style spatial systems

---

## 1. Linear Sequence (Progression)

**Use when:** order matters (steps, logic flow)

**Structure:**

* Horizontal or vertical progression
* Each item isolated
* Equal spacing, but not boxed

**Rules:**

* No paragraphs
* Each step = one idea
* Visual rhythm > decoration

---

## 2. Causal Chain (3-Zone Logic)

**Use when:** cause → mechanism → outcome

**Structure:**

* 3 zones (not equal)
* Left: premise
* Center: system/mechanism
* Right: consequence

**Rules:**

* Center is dominant
* Side zones lighter
* Never collapse into text blocks


---

## 3. Asymmetric Split (Tension Layout)

**Use when:** contrast, conflict, or imbalance

**Structure:**

* 30/70 or 40/60 split
* One side carries weight
* Other side reframes or challenges

**Rules:**

* Never 50/50
* One side must feel “heavier”

---

## 4. Principle Stack (Declarative System)

**Use when:** foundational truths / rules

**Structure:**

* Vertical stack
* Each line isolated
* Large spacing between items

**Rules:**

* No grouping into paragraphs
* Each line must stand alone

This is very “consulting memo” style.

---

## 5. Framed Core + Peripheral Context

**Use when:** one central idea + supporting signals

**Structure:**

* Center: dominant statement
* Surrounding: annotations / supporting lines

**Rules:**

* Center must visually dominate
* Peripheral content is lighter, smaller

---

## 6. Indexed Narrative (Guided Reading)

**Use when:** structured explanation with navigation

**Structure:**

* Left: vertical index (labels/numbers)
* Right: flowing content

**Rules:**

* Index is static anchor
* Content is dynamic
* Avoid turning content into a text wall

---

## 7. Parallel Axes (Multi-Dimension View)

**Use when:** multiple equal perspectives

**Structure:**

* 3–4 columns or rows
* Each represents one dimension

**Rules:**

* All items equal weight
* No hierarchy inside grid
* Clean, sharp alignment

---

## 8. Compression Block (Density Control)

**Use when:** heavy content that must not feel heavy

**Structure:**

* Narrow width
* Controlled line length
* Strong spacing rhythm

**Rules:**

* Never full-width paragraphs
* Typography carries structure

---






# The mapping (this is critical)



| Thinking Model | Spatial Pattern                   |
| -------------- | --------------------------------- |
| Sequence       | Linear progression                |
| Causality      | 3-zone (cause → system → outcome) |
| Constraint     | Framed or isolated block          |
| Contrast       | Asymmetric split                  |
| Principle      | Vertical stack                    |
| System         | Indexed or structured layout      |
| Evaluation     | Grid / parallel axes              |
| Transformation | Before/after split                |





# Rules

---

## Rule 1 — One section = one pattern

Do NOT mix patterns inside a section.

That’s how things become messy and “meaningless.”

---

## Rule 2 — No consecutive repetition

If you use:

* Causal Chain

Next section CANNOT be:

* Causal Chain again

You must rotate:
→ Sequence
→ Split
→ Stack
→ etc.

---

## Rule 3 — Kill symmetry by default

If something looks:

* balanced
* centered
* evenly distributed

…it’s probably wrong for this style.

# Fixed style rules


## 1. Layout & Background — Non-Negotiables

### A. Screen Width (Structural Requirement)

* The interface must operate in a **wide-screen layout by default**
---

### B. Background System (Strictly Limited)

Allowed backgrounds:

* **Navy Blue**
* **White**
* **One optional neutral (light cool gray only)**

---

### C. Background Roles (Functional Mapping)

* **Navy Blue**

  * Used for **high-weight sections**
  * Signals **system importance or conceptual emphasis**
  * Typically paired with **light typography**

* **White**

  * Default surface for **primary content**
  * Used for **clarity, readability, and decomposition**

* **Light Cool Gray (optional)**

  * Used only when **white-to-white transitions reduce clarity**
  * Acts as a **separation layer**, not a design feature
  * Must remain **visually subtle and low contrast**
